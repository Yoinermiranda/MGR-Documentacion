19. Prototipo e Interfaz

El prototipo del sistema MGR fue desarrollado con el propósito de representar la estructura visual, navegación y comportamiento esperado de la aplicación antes de su implementación final.

La interfaz permite validar la organización de los módulos, distribución de elementos visuales y experiencia de usuario, garantizando que el sistema sea intuitivo, funcional y adaptado a las necesidades del usuario final.

19.3 Descripción de la Interfaz

Pantalla de Acceso

La interfaz de autenticación fue diseñada para brindar un acceso seguro y sencillo al sistema, permitiendo que los usuarios ingresen mediante un proceso claro y organizado.

Panel Principal

El dashboard presenta una estructura organizada donde el usuario puede acceder rápidamente a los diferentes módulos del sistema y consultar información relevante.

Módulos del Sistema

Las interfaces de los módulos fueron diseñadas mediante componentes visuales, formularios y tablas, facilitando la gestión de información y las operaciones principales del aplicativo.

19.4 Justificación de Decisiones UX/UI

  • Organización de la información: La interfaz utiliza una estructura modular que permite encontrar las funcionalidades de manera rápida y ordenada.
  • Simplicidad visual: Se aplicó un diseño limpio evitando elementos innecesarios para mejorar la comprensión y navegación del usuario.
  • Consistencia del diseño: Los componentes visuales mantienen estilos uniformes en botones, formularios, menús y secciones del sistema.
  • Facilidad de uso: La distribución de los elementos fue pensada para reducir la cantidad de pasos necesarios para ejecutar las acciones principales.
  • Experiencia del usuario: El diseño prioriza una interacción clara, rápida y eficiente con el sistema.

19.5 Resultado del Prototipo

El prototipo desarrollado permite comprobar la funcionalidad visual del sistema MGR, validando la navegación entre pantallas, estructura general y experiencia del usuario antes de la versión final.
